# Proof Boundaries

These recipes are intentionally lookup and memory-emission tools only.

They can support a generalized viral source-edit route by surfacing:

- edit-to-source candidate and confirmed moments,
- recurring source moments and transcript-backed story patterns,
- broad source usage rows that are good planning priorities,
- sequence-level story functions,
- song, beat, and action timing candidates.

They cannot prove new source matches, promote candidates, correct human labels, or claim frame-accurate source reconstruction. Any downstream editor must keep source timestamps, reference edit timestamps, and derived planning recommendations separate.

When writing learned-style memory, the output must carry the boundary that `preferred` means "planning priority" only. It does not convert a candidate row into a confirmed row.
